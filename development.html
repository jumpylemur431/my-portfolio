Problem

Context

This application allows users to buy albums by GOT7, a South Korean K-pop boy group. The goal is to allow users to browse all released albums by the group and buy them. The value of the website is to allow users or GOT7 fans to have a one-stop shop to buy all the albums or EPs they would like, with detailed sorting and filtering on each item.

Why React?

This application is an exploration in using React, a JavaScript library for building user interfaces quickly and efficiently. In React, users can develop applications by creating reusable components. Therefore, multiple components such as cards and cart items are re-used in this React app, all while developed in JavaScript and CSS.

React Development

React Usability Principles

React usability principles were incorporated in the filters and sorts I included on the page. For general users who would like to browse albums, there are sorts for the price and popularity of the albums. For more serious K-pop or GOT7 fans, they can learn further details on the album such as the lead single producer or the number of songs on an album. The filters lie on the left of the page with the cart on the right to view all major components of the page at once.

State Changes

For component state changes, there five states that change. The first is sorting that is updated as a new sort type is selected. Next is the filters that updates when a certain filter is checked in the check-mark box. Next is the cart that is updated with albums as they are added to the cart. Similarly, the total amount is updated with an integer value depending on item price. Finally, the cart and filters can be cleared or reset as needed which changes state.

Design Iterations

Low Fidelity Mock-up

The following low fidelity mock-up covers the major parts of the React web application. There are three main divs that split the page by the ratio 20:60:20.

The largest div includes the album cards. These will include the album cover image, the name, and additional information that the albums are sorted by. These cards are dynamically changing depending on the filters selected by users

The left-most div includes all the filters and sorting options. First are sorting options in the form of radio buttons (not pictured in the design). The second are the two filters: by album type and producer in the form of checkboxes. There is also a reset button to reset all filters and sorts.

The right-most div is the cart for users to add each album to. The cart total updates as users add more items to the cart. The count of each album can also be updated in the cart to include more or less depeding on the users preference. Finally, they can checkout with the selected album names once they have completed shopping. They can also clear the cart to resert shopping.

Conclusion

Insights

In this project, I learned how exactly React works in building responsive and interactive webpages. I built my JavaScript knowledge through this project and found the value of building webpages with React.

Takeaways

A takeaway I had in this project was React components. With React components, since they are reusable, time is saved as I do not need to write different code for the same features. Additionally, if I make any changes to a certain part, it does not affect other components in the application.

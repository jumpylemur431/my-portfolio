Design Question

Context

User Experience is a fundamental aspect of product design. To understand the value of user experience this project reviews a common interface a digital payment platform: Toast Point of Sale (POS) Platform. In this review, I observered and interviewed users of the interface at Charuma Tea Bar. In order to know the users of the product well, UX designers make user personas and storyboard the entire user experience. Therefore, for this review, I desgined two personas and a storyboard based on my UX research.

Why?

In most cafes, there are multiple ways cafe-goers pay for their drinks or snacks. I, for instance, tend to use my iPhone Apple Pay method but other cafe-goers use their card chip, tap, or swipe capabilities. There are also capabilities to view the order amount, add tip, join rewards programs, and receive the receipt that adds to the user experience when paying for a drink or snack at the cafe. Therefore, this project allowed for a deeper exploration of a cafe system users, and myself, interact with everyday.

User Research

User Observations

I observered three different users each and the actions they took when interacting with the Toast Interface. From each user, I made object observations of their interation, from how long the experience took, the payment method selected, to how many times they touched the screen. Their unique experiences helped me better understand how users broadly interact with the interface and user patterns. This process took around an hour at Charuma Tea Bar.

User Interviews

I also conducted interviews with 10 of the users of the Toast Interface within a three hour time span. I asked users to answer questions in regards to their experience with the interface and their motivations to use the interface in certain ways. By asking these questions, I gained first hand thoughts and feelings from users to better understand what the user is considering when interacting with the interface.

User Results

Personas

Using the data collected from observations and interviews, I created two user personas. The personas are based on overarching trends observed amongst all the users. I created two personas, Sam and Lisa. Both Lisa and Sam, although different experiences, represent two types of users who interact with the Toast platform: the confident user and the confused user. To further build the personas, I created a four-quadrant empathy map describing what each persona thinks, feels, says, and does with the platform.

Storyboarding

After designing personas, I created a storyboard for their user journey from start to end. I took the persona Lisa and reflected their experience using the Toast interface. The storyboard reflects the goals of the persona and includes all necessary six steps for using the interface.

Conclusion

Insights

In this project, I learned the importance of conducting user experience research. Simply assuming what the user wants in a product will not produce a simple product. Instead, going to the user and observering their actions and asking questions will yield to interesting insights that can be useful in designing a better product user experience.

Takeaways

A takewaway I had for this project is how developing personas and storyboarding can more clearly build a user story. There are so many users that may use a product but telling the story data provides with an empathy map or a storyboard clarifies what their needs, wants, and frustrations are that is invaluable information to the product designer.
